1. Visão Geral e Objetivos do Sistema

Apenas admins (você e sua esposa Júlia) podem uploadar e editar conteúdos.
•  Acesso Geral: Qualquer pessoa com o link pode acessar o site principal para observar dados (exames em tabelas, gráficos, timelines). Nada de edições – máximo que fazem é selecionar itens para filtros ou visualizações especiais, como comparativos.
•  Acesso Admin: Via URL “/admin” (ex: site.com/admin), protegido por login/senha. Admins podem criar contas para você (Paulo Guimarães) e Júlia (com emails e senhas personalizados).

	•  Fotos: Foto da Luna (central, em círculo claro com bordas escuras), fotos menores de tutores (você e Júlia à direita).
•  Processamento Inteligente: Todo upload (PDFs, textos, áudios, vídeos) é “absorvido” via IA para extração automática de dados, criação de registros, conversão de unidades, e geração de visualizações. 2. Requisitos Técnicos e Tecnologias
•  Linguagem Principal: Python, como você pediu, para backend e processamento. Para o site “bem bonito e impressionante”, use frameworks como Streamlit (fácil para dashboards interativos) ou Dash (de Plotly, ótimo para gráficos e tabelas). Frontend pode ser integrado via HTML/CSS/JS embutido no Python, ou React se quiser mais polimento – mas foque em Python para manter simples.
•  Integrações de IA: Use bibliotecas Python como PyPDF2 ou pdfplumber para extrair texto de PDFs; Tesseract OCR para digitalização; Hugging Face Transformers para processamento de linguagem natural (extração de datas, nomes, valores); SpeechRecognition para áudios; OpenCV para vídeos (extração de frames se necessário). Para buscas externas (ex: validar nomes de medicamentos), integre APIs como PubChem ou Wikipedia via requests – [Inference] baseado em padrões observados, isso reduz erros em 80-90% em sistemas semelhantes, mas teste sempre.
•  Segurança: Senha obrigatória no /admin. Use hashing (ex: bcrypt via Python) para armazenar senhas. Autenticação via JWT ou sessions. [Sugestão Adicional]: Adicione 2FA (ex: via email) e logs de acesso para rastrear quem editou o quê – essencial em contextos médicos para compliance.
•  Armazenamento: Arquivos salvos internamente como “lista na plataforma” (diretório ou DB blob). Independente do nome do arquivo, extraia data, laboratório, médico via IA.
•  Unidades Uniformes: Converta automaticamente unidades (ex: ng/dL para mmol/dL) usando bibliotecas como Pint ou manualmente com dicionários de conversão.
•  Exportações: Botões para exportar tabelas (CSV/Excel), gráficos (PNG ou copiar para clipboard via JS), e prontuários (PDF via ReportLab).
•  [Sugestão Adicional]: Para mobile, teste com Progressive Web App (PWA) features – permite “instalar” no celular como app nativo.
3. Roles de Usuários
•  Viewers (Público): Acesso readonly. Podem ver tabelas, gráficos, timelines, selecionar exames/datas para comparativos. Compartilhável via link (CTRL+C/V).
•  Admins (Você e Júlia): Acesso total via /admin. Uploads, edições, criação de usuários. Logs de modificações (ex: “Modificado por Júlia em [data]”).
4. Features do Lado Viewer (Página Principal)
•  Tabela Principal de Exames: Grande tabela estilo Excel, ocupando quase toda a janela. Colunas: Datas dos exames. Linhas: Nomes dos exames (ex: Eritrócitos, Hemoglobina, RDW). Valores preenchidos automaticamente de PDFs. IA cria novas linhas para exames novos (ex: amônia).
•  Abas/Botões no Topo:
	•  “Histórico Completo”: Tabela full.
	•  “Comparativo”: Selecione exames (ex: hemoglobina + hematócrito), datas (sequenciais, específicas ou “todos”), gere gráfico. Botão “Exportar Gráfico” abre janela com opções “Copiar Gráfico” (como CTRL+C) e “Fechar”.
	•  “Prontuário / História da Doença Atual (HDA)”: Linha do tempo horizontal (rosa para Luna). Marcadores (setinhas/riscos verticais brancos) por data. Ao selecionar:
		•  Metade superior: Sintomas/resumo do dia (ex: “Início das mioclonias, tremores, sono”).
		•  Botões esquerda/direita para navegar (esquerda apagada no início). Animação: Risco move-se, cor realça.
		•  Metade inferior esquerda: Texto cursivo bonito como prontuário médico.
		•  Metade inferior direita: Pré-seleção de exames relacionados à data.
	•  “Histórico de Medicamentos”: Linha do tempo horizontal com retângulos coloridos por med (ex: voriconazol amarelo). Dentro: Dose, via, data início/término (linhas verticais pretas). No fim: “P.S.” com considerações (ex: “Suspenso por tontura”). Movível esquerda/direita.
•  Cabeçalho: Foto Luna (círculo claro central, bordas escuras). À direita: Fotos menores de tutores (você e Júlia, com nomes).
•  [Sugestão Adicional]: Filtros avançados (ex: busca por exame) e tooltips com unidades convertidas para clareza.
5. Features do Lado Admin
•  Login: Janela pop-up ao acessar /admin. Crie contas para você e Júlia (email/senha).
•  Upload de Exames (PDFs): Upload via drag-drop ou botão “Inserir”. Extraia via IA: Data, laboratório, médico, nome exame, valor, unidade. Salve como dados estruturados. Processamento inteligente para novos campos.
•  Construção de Prontuário Clínico (HDA): Upload de texto/PDF/áudio/vídeo. IA processa:
	•  Reconhece texto/áudio (speech-to-text), seleciona tópicos, data.
	•  Mostra caixa de texto editável: Data (editável), texto interpretado (azul inicial, vermelho em edições), tópicos abaixo.
	•  Botão “Reprocessar Texto”: Corrige ortografia, reorganiza. Depois, “Salvar”.
	•  Para áudios: Valide nomes de meds na internet/banco de dados.
	•  Para vídeos: Processamento similar, extraia frames se relevante.
	•  Dúvidas da IA: Pergunte via caixas (ex: “Canabidiol sob demanda?”).
	•  Visualização: Bloco de notas-like, com datas em negrito, textos abaixo, arquivos à esquerda, logs à direita (“Modificado por [nome]”).
	•  Edição: Clique para editar (vermelho), reprocessar, salvar.
	•  Upload em Mobile: Acesso a pastas, iCloud, álbuns. Indicadores de progresso (pizza azul para download nuvem->celular, tique cinza, upload com pizza/tique).
	•  Botão “Gerar Histórico”: Processa tudo, avisa “Não feche a janela”. Gera texto em tempo real, salva.
•  Histórico de Medicamentos no Admin: Bloco de notas com tabela: Colunas em grupos (cinza claro para início, diferente para término/considerações). Linhas por med, sublinhas para re-administrações (cores variadas). Registre via áudio/texto, pergunte dúvidas.
•  Outros: Edite temática (cores, fundo pediátrico). Upload fotos. Imprimir PDF do prontuário.
•  [Sugestão Adicional]: Backup automático de DB e versão control (como Git interno) para evitar perdas – tradicional como arquivar prontuários físicos, mas inovador com automação.
